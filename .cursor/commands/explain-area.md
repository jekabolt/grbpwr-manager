# Explain a codebase area

Given a package or directory path, provide:

1. **Purpose** — what this package does in the system
2. **Key types** — main structs, interfaces, and their roles
3. **Dependencies** — what it imports from other internal packages
4. **Data flow** — how data enters and exits this package
5. **Entry points** — exported functions/methods that callers use
6. **Configuration** — any config this package reads
7. **Related packages** — other packages that work alongside this one

Reference @000-project-overview.mdc for overall architecture context.
Keep the explanation concise but cover the non-obvious parts.
